# Quick Start

```python

Python Libraries
pypdf — Basic PDF Operations
Merge PDFs
from pypdf import PdfReader, PdfWriter

writer = PdfWriter()

for pdf_file in ["doc1.pdf", "doc2.pdf", "doc3.pdf"]:
    reader = PdfReader(pdf_file)

    for page in reader.pages:
        writer.add_page(page)

with open("merged.pdf", "wb") as output:
    writer.write(output)
Split PDF into Individual Pages
from pypdf import PdfReader, PdfWriter

reader = PdfReader("input.pdf")

for i, page in enumerate(reader.pages):
    writer = PdfWriter()
    writer.add_page(page)

    with open(f"page_{i + 1}.pdf", "wb") as output:
        writer.write(output)
Extract Metadata
from pypdf import PdfReader

reader = PdfReader("document.pdf")
meta = reader.metadata

print(f"Title: {meta.title}")
print(f"Author: {meta.author}")
print(f"Subject: {meta.subject}")
print(f"Creator: {meta.creator}")
Rotate Pages
from pypdf import PdfReader, PdfWriter

reader = PdfReader("input.pdf")
writer = PdfWriter()

page = reader.pages[0]
page.rotate(90)

writer.add_page(page)

with open("rotated.pdf", "wb") as output:
    writer.write(output)

pdfplumber — Text and Table Extraction

import pdfplumber

Extract Tables
import pdfplumber

with pdfplumber.open("document.pdf") as pdf:
    for page_num, page in enumerate(pdf.pages, start=1):
        tables = page.extract_tables()

        for table_num, table in enumerate(tables, start=1):
            print(f"Table {table_num} on page {page_num}")

            for row in table:
                print(row)
Export Tables to Excel
import pandas as pd
import pdfplumber

all_tables = []

with pdfplumber.open("document.pdf") as pdf:
    for page in pdf.pages:
        tables = page.extract_tables()

        for table in tables:
            if table:
                df = pd.DataFrame(table[1:], columns=table[0])
                all_tables.append(df)

if all_tables:
    combined_df = pd.concat(all_tables, ignore_index=True)
    combined_df.to_excel("extracted_tables.xlsx", index=False)

reportlab — Create PDFs
Create a Simple PDF
from reportlab.lib.pagesizes import letter
from reportlab.pdfgen import canvas

c = canvas.Canvas("hello.pdf", pagesize=letter)

width, height = letter

c.drawString(100, height - 100, "Hello World!")
c.drawString(100, height - 120, "This is a PDF created with reportlab")

c.line(100, height - 140, 400, height - 140)

c.save()
Create a Multi-Page PDF
from reportlab.lib.pagesizes import letter
from reportlab.platypus import (
    SimpleDocTemplate,
    Paragraph,
    Spacer,
    PageBreak,
)
from reportlab.lib.styles import getSampleStyleSheet

doc = SimpleDocTemplate("report.pdf", pagesize=letter)

styles = getSampleStyleSheet()
story = []

story.append(Paragraph("Report Title", styles["Title"]))
story.append(Spacer(1, 12))

body = Paragraph(
    "This is the body of the report. " * 20,
    styles["Normal"],
)

story.append(body)
story.append(PageBreak())

story.append(Paragraph("Page 2", styles["Heading1"]))
story.append(Paragraph("Content for page 2", styles["Normal"]))

doc.build(story)
Subscripts and Superscripts
Do not use Unicode subscript or superscript characters in ReportLab PDFs:
₀₁₂₃₄₅₆₇₈₉
⁰¹²³⁴⁵⁶⁷⁸⁹
Many built-in fonts do not support these glyphs.
Use ReportLab XML tags instead:
from reportlab.platypus import Paragraph
from reportlab.lib.styles import getSampleStyleSheet

styles = getSampleStyleSheet()

chemical = Paragraph(
    "H<sub>2</sub>O",
    styles["Normal"],
)

equation = Paragraph(
    "x<super>2</super> + y<super>2</super>",
    styles["Normal"],
)
For canvas-based drawing, manually adjust font size and positioning.

OCR for Scanned PDFs
import pytesseract
from pdf2image import convert_from_path

images = convert_from_path("scanned.pdf")

text = ""

for i, image in enumerate(images, start=1):
    text += f"Page {i}\n"
    text += pytesseract.image_to_string(image)
    text += "\n\n"

# XLSX Creation, Editing, and Analysis

## Core Requirements

### Professional Formatting
- Use a consistent professional font unless the user specifies otherwise
  - Preferred: Arial or Times New Roman
- Maintain clean spacing and alignment

### Zero Formula Errors
All delivered Excel models must contain:
- No `#REF!`
- No `#DIV/0!`
- No `#VALUE!`
- No `#N/A`
- No `#NAME?`

### Preserve Existing Templates
When editing existing workbooks:
- Match the workbook’s existing style and conventions exactly
- Preserve established formatting patterns
- Do not impose new formatting standards unless requested
- Existing template conventions override these guidelines

# Financial Model Standards

## Color Coding Conventions

Unless overridden by the workbook template or user instructions:

| Purpose | Format |
|---|---|
| Hardcoded inputs | Blue text (RGB 0,0,255) |
| Formulas/calculations | Black text (RGB 0,0,0) |
| Internal worksheet links | Green text (RGB 0,128,0) |
| External workbook links | Red text (RGB 255,0,0) |
| Key assumptions | Yellow fill (RGB 255,255,0) |

## Number Formatting

| Type | Standard |
|---|---|
| Years | Text format (`2024`, not `2,024`) |
| Currency | `$#,##0` |
| Percentages | `0.0%` |
| Multiples | `0.0x` |
| Negative numbers | `(123)` not `-123` |
| Zeros | Display as `-` |

Examples:
- Currency: `$#,##0;($#,##0);-`
- Percentages: `0.0%;(0.0%);-`

Always include units in headers:
- Example: `Revenue ($mm)`

# Formula Construction Rules

## Assumptions
- Store assumptions in dedicated cells
- Reference assumption cells in formulas
- Never hardcode assumptions inside formulas

Correct:
```excel
=B5*(1+$B$6)
Incorrect:
=B5*1.05
Formula Quality Checks
* Verify all references
* Avoid off-by-one errors
* Keep formulas consistent across periods
* Test zero and negative-value edge cases
* Avoid unintended circular references
Hardcoded Input Documentation
Document all hardcoded assumptions or imported values.
Format:
Source: [System/Document], [Date], [Reference], [URL if applicable]
Examples:
* Source: Company 10-K, FY2024, Page 45
* Source: Bloomberg Terminal, 8/15/2025
* Source: FactSet, 8/20/2025

Python Workflow Guidance
Recommended Libraries
Library	Best Use
pandas	Data analysis, cleaning, bulk operations
openpyxl	Formatting, formulas, Excel-specific editing
CRITICAL: Use Excel Formulas
Always write calculations as Excel formulas.
Do NOT calculate values in Python and hardcode results.
Incorrect:
total = df['Sales'].sum()
sheet['B10'] = total
Correct:
sheet['B10'] = '=SUM(B2:B9)'
The workbook must remain dynamic and recalculate correctly when source values change.

Standard Workflow
1. Load or create workbook
2. Modify data, formulas, and formatting
3. Save workbook
4. Recalculate formulas
5. Validate and fix errors
Mandatory Formula Recalculation

python scripts/recalc.py output.xlsx

* Recalculates formulas using LibreOffice
* Detects Excel errors
* Returns structured JSON output
* Works on Linux and macOS
LibreOffice setup is handled automatically.

Formula Verification Checklist
Essential Checks
* Verify sample references before scaling formulas
* Confirm correct column mappings
* Remember Excel uses 1-based indexing
Common Pitfalls
* Null/NaN handling
* Wrong sheet references
* Division by zero
* Missing referenced cells
* Incorrect far-right column mapping
Testing Strategy
* Start with a few cells
* Verify dependencies
* Test edge cases:
    * Zero values
    * Negative values
    * Large values

Interpreting recalc.py Output
Example:
{
  "status": "success",
  "total_errors": 0,
  "total_formulas": 42
}
If errors exist:
{
  "status": "errors_found",
  "error_summary": {
    "#REF!": {
      "count": 2,

    }
  }
}

Best Practices
openpyxl
* Excel indexing is 1-based
* Use data_only=True only when reading calculated values
* Saving a workbook opened with data_only=True removes formulas
* Use read_only=True for large reads
* Use write_only=True for large writes
pandas
* Specify dtypes when needed
* Limit columns for large files
* Parse dates explicitly
Example:
pd.read_excel(
    'file.xlsx',
    dtype={'id': str},
    parse_dates=['date']
)

Code Style
Python Code
* Keep code concise
* Avoid unnecessary comments
* Avoid redundant variables
* Avoid excessive print statements
Excel Models
* Add comments for complex formulas
* Document assumptions
* Add notes for key calculations

Design Guidance
Core Principles
Avoid Generic Slides
Do not create plain bullet lists on white backgrounds. Every slide should feel intentionally designed.
Use a Topic-Specific Color Palette
Choose colors that match the subject matter. If the same palette works equally well for unrelated topics, it is probably too generic.
Establish Visual Hierarchy
* One dominant color should carry 60–70% of the visual weight
* Use 1–2 supporting colors
* Reserve a strong accent color for emphasis
Maintain Consistent Visual Motifs
Pick one reusable design element and repeat it throughout the deck, such as:
* Rounded image frames
* Colored icon circles
* Thick side borders
* Consistent card treatments
Consider a “Sandwich” Structure
A common pattern:
* Dark title slide
* Light content slides
* Dark conclusion slide
Or commit fully to a dark theme for a premium look.

Tables
Critical Rules
* Always use WidthType.DXA
* Never use percentages
* Table width must equal the sum of column widths
* Set widths on:
    * table
    * columns
    * cells
Example

  style: BorderStyle.SINGLE,
  size: 1,
  color: "CCCCCC",
};

const borders = {
  top: border,
  bottom: border,
  left: border,
  right: border,
};

new Table({

Critical Rules
Document Structure
* Set page size explicitly
* Use separate Paragraph elements instead of \n
* PageBreak must be inside a Paragraph
Lists
* Never manually insert bullet characters
* Use numbering config with LevelFormat.BULLET
Tables
* Always use WidthType.DXA
* Never use percentage widths
* Table width must equal column widths
* Cell widths must match columns
* Always set cell margins
* Use ShadingType.CLEAR
Images
* Always specify image type
TOC & Headings
* Use HeadingLevel
* Override built-in style IDs:
    * Heading1
    * Heading2
* Include outlineLevel

Editing Existing Documents
Workflow
1. Unpack
2. Edit XML
3. Repack

Step 1 — Unpack
python scripts/office/unpack.py document.docx unpacked/
Features:
* Pretty-prints XML
* Merges adjacent runs
* Converts smart quotes to XML entities

Step 2 — Edit XML
Edit files in:
unpacked/word/
Rules
* Use "Claude" as author unless specified otherwise
* Use direct edits instead of scripts
* Preserve formatting runs
* Use smart quotes
Smart Quote Entities
Entity	Character
&#x2018;	‘
&#x2019;	’
&#x201C;	“
&#x201D;	”
Example:
<w:t>Here&#x2019;s a quote: &#x201C;Hello&#x201D;</w:t>

Adding Comments
python scripts/comment.py unpacked/ 0 "Comment text"
Reply:
python scripts/comment.py unpacked/ 1 "Reply text" --parent 0
Custom author:
python scripts/comment.py unpacked/ 0 "Text" --author "Custom Author"

Step 3 — Repack
python scripts/office/pack.py unpacked/ output.docx --original document.docx

XML Reference
Schema Rules
* <w:rPr> must appear last in <w:pPr>
* Use xml:space="preserve" when needed
* RSIDs must be 8-digit hex values

Tracked Changes
Insertion
<w:ins w:id="1" w:author="Claude" w:date="2025-01-01T00:00:00Z">
  <w:r>
    <w:t>inserted text</w:t>
  </w:r>
</w:ins>
Deletion
<w:del w:id="2" w:author="Claude" w:date="2025-01-01T00:00:00Z">
  <w:r>
    <w:delText>deleted text</w:delText>
  </w:r>
</w:del>

Minimal Edits
Only mark changed text.
<w:r><w:t>The term is </w:t></w:r>

<w:del w:id="1" w:author="Claude">
  <w:r><w:delText>30</w:delText></w:r>
</w:del>

<w:ins w:id="2" w:author="Claude">
  <w:r><w:t>60</w:t></w:r>
</w:ins>

<w:r><w:t> days.</w:t></w:r>

Comments
commentRangeStart and commentRangeEnd must be siblings of w:r.
<w:commentRangeStart w:id="0"/>

<w:r>
  <w:t>text</w:t>
</w:r>

<w:commentRangeEnd w:id="0"/>

<w:r>
  <w:rPr>
    <w:rStyle w:val="CommentReference"/>
  </w:rPr>

  <w:commentReference w:id="0"/>
</w:r>
